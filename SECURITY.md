# Security

## Reporting

If you discover a security issue in Sah.Bukti, do not open a public issue with exploit details.

Report it privately to the repository owner first with:

- affected area
- reproduction steps
- expected impact
- any suggested remediation

## Product Security Principles

- Approval is the only path that mutates ledger truth
- Payment-like evidence is review-gated
- Public demo surfaces must not expose private connector state, owner numbers, or secret-bearing setup controls
- Secrets and live credentials must not be committed to the repository
