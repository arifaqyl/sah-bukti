# Sah.Bukti Pivot Plan

Generated: 2026-06-16

## New Core Thesis

The strongest 4-day hackathon wedge is no longer:

- screenshot-first payment verification

It is now:

- reviewable evidence intake
- payment link generation
- webhook payment confirmation
- small-seller ledger

## Why

- low-ticket Malaysian sellers dislike flat gateway fees
- but screenshot verification is still too brittle as a primary trust layer
- payment links with webhook confirmation produce a much cleaner demo
- this still fits Malaysia and Kracked Devs better than a generic AI app

## Product Position

`Sah.Bukti` is not another gateway.

It is:

- the workflow layer for Malaysian sellers who already sell through WhatsApp
- the thing that turns a messy chat sale into a structured paid order

## Build Direction

### Core now

1. create invoice from chat-like input
2. generate payment link
3. share on WhatsApp
4. receive webhook confirmation
5. update ledger automatically

### Keep but demote

- receipt screenshot verification
- static QR page
- soundbox-like ideas

Those are now secondary or future lanes, not the main story.

## Suggested Segment

- home bakers
- Instagram sellers
- WhatsApp catalog sellers
- freelancers with simple invoice needs

Avoid centering:

- pasar stalls with RM5 items

because the fee economics are worse there and the main story gets muddier.
