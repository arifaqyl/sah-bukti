# Sah.Bukti Taste System

## Direction

Sah.Bukti is not a fintech dashboard. It is a warm, serious operations notebook for real shop owners who live in WhatsApp.

The interface should feel like:
- a handwritten ledger
- a clean café receipt
- a calm operations console
- human, local, trustworthy

It should not feel like:
- AI SaaS template
- crypto dashboard
- generic indigo startup app
- neon dark-mode admin panel
- corporate fintech clone

## Palette

Primary anchor: clay red `#B85C38`.
Paper: warm cream `#F7F0E6`.
Ink: soft black brown `#171412`.
Success: sage green `#6F8F72`.
Warning: turmeric yellow `#B7791F`.
Danger: rust red `#C94B35`.
Info: rain blue grey `#5F7F91`.

No purple. No blue gradients. No neon. No rainbow badges. No pastel overload.

## Typography

Use system serif for display moments and system sans for operational text.
Display type should feel editorial and human.
Operational text should be compact and readable.

## Layout

Desktop:
- paper background
- dark espresso sidebar
- cream cards
- clay primary actions
- sage/rust/turmeric semantic states

Mobile:
- sidebar becomes a compact drawer
- cards stack
- tables scroll
- forms use full width

## Component taste

Cards:
- soft paper surfaces
- visible borders
- generous radius
- no heavy shadows
- no random gradients

Buttons:
- pill-shaped
- clay primary
- quiet secondary
- strong hover state

Badges:
- semantic colors only
- small, dense, readable

Empty states:
- useful explanation
- one clear action
- no vague copy

## Material names

Use these names in code, PRs, and chat:
- paper
- ink
- clay
- sage
- rail
- stack
- bento
- checklist

## Motion taste

Keep motion light and user-like:
- floatIn for cards and panels
- pulseLine for flow indicators
- soft drift for phone hero
- stagger for list reveals

Avoid brassGlow and any heavy/bright glow effects.
Prefer CSS-only materials. No normal SVG.

## Anti-slop rules

- No lorem ipsum.
- No purple/indigo default SaaS palette.
- No decorative blobs.
- No fake AI claims.
- No oversized marketing cards inside product screens.
- No vague phrases like "unlock insights" or "next-generation".
- Every page must help the owner make a decision.
